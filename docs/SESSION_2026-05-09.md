Crucial work-arounds learned and documented:
- `dual_chunk_attention_config` removed from Qwen's `config.json` — vLLM v1 has no DCA-aware impl, so the unmodified config crashes `qwen2.py` at init with `TypeError: FlashAttentionImpl.__init__() got an unexpected keyword argument 'layer_idx'`.
- `sparse_attention_config.json` renamed to `.disabled` for the same chain (loaded only when DCA present).
- `enforce_eager=True` because CUDA graphs segfault on sm_86 + FLASHINFER on long sequences.
- `VLLM_MEMORY_PROFILER_ESTIMATE_CUDAGRAPHS=0` so vLLM's CUDA-graph memory reservation doesn't eat into KV.

### PR1a — AWQ INT4 weights
Pivot away from FP8: `kv_cache_dtype=fp8` and `cortecs/Qwen2.5-7B-Instruct-1M-FP8-Dynamic` both produce gibberish — Marlin FP8 emulation on Ampere is broken for inference (vLLM only warns about perf, but the symptom is corruption). `graelo/Qwen2.5-7B-Instruct-1M-AWQ` works:

| ctx | mem (GB) | t/s | needle |
|-----|----------|-----|--------|
|  4K | 21.73    | 3,272 | ✓ |
| 32K | 22.34    | 3,450 | ✓ |
|128K | 22.34    | 1,822 | ✓ |

3.5 GB weights instead of 14 GB freed enough VRAM that 128K runs on **one** 3090, leaving GPU 1 free for PR5.

### PR1b — nuq KV math/scales/kernels validated
Three calibration sweeps:
| sweep | num_bits | outlier_pct | needle      |
|-------|----------|-------------|-------------|
| v1    | 2        | 0.005       | FAIL `'7392!B! The!! 1392!'` |
| v2    | 2        | 0.02        | FAIL `' BAN!@!@!@!@!@!@'`    |
| v3    | **4**    | 0.02        | **PASS** `' BANANA!7!3!9!2!'` |

Layer 27 (last) Values mean recon error: 1.40 → 0.97 → **0.18**. Bottom line: Qwen-7B late-layer V's need 4-bit (16 levels) without the sparse-outlier kernels from the upstream KVQuant paper; nuq2 alone is insufficient.

## What's deferred and why
- **PR2 (MInference Vertical-Slash)**: vLLM 0.20.1 lacks the merged MInference kernels (they're in ≥0.21), and the Qwen sparse pattern config requires DCA which v1 dropped. Three full investigation paths in the commit log; punted until vLLM upgrade.
- **PR1c / nuq2 + sparse outliers**: needs the sparse-outlier kernel from the original KVQuant paper for last-layer V; not justified before PR5.
- **vLLM integration of nuq4** (the inference-time payoff of PR1b): documented as `~2 weeks of plumbing, no new research`.
